feat(mqtt): simplify implementation and optimize for mobile-only support

This commit finalizes the transition from a complex platform-agnostic MQTT 
factory to a streamlined, native-first architecture (Android & iOS).

Changes:
- Refactored `MqttClientManager` to use `MqttServerClient` directly, reducing 
  abstraction overhead and platform-specific setup complexity.
- Implemented SSL certificate bypass (`onBadCertificate`) in `MqttClientManager` 
  to resolve connection issues with HiveMQ Cloud on mobile devices.
- Removed all platform-agnostic setup files (`mqtt_setup.dart`, etc.) and 
  conditional imports throughout the `lib/` directory.
- Scrubbed `kIsWeb` flags and unused platform-specific logic from 
  `MqttDataSource` and `firebase_options.dart`.
- Deleted redundant platform directories for `web/`, `windows/`, `macos/`, 
  and `linux/` to optimize the workspace.
- Updated `pubspec.yaml` to remove web dependencies and platform-specific configs.
- Restored missing `MonitorRepository` dependency in `main.dart` to fix UI 
  initialization errors.

These changes ensure a more stable and maintainable codebase focused on 
native mobile performance.
